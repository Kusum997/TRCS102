# JavaScript File (practice.js)


# JavaScript Naming Conventions

Good variable names make the code readable and easier to maintain.

## Camel Case

Used for variables and functions.

```javascript
let firstName;
let studentAge;
let totalMarks;
```

---

## Pascal Case

Used for classes and constructor functions.

```javascript
class StudentData{}
class UserProfile{}
```

---

## Snake Case

Words are separated using underscores.

```javascript
let first_name;
let total_marks;
```

---

## Kebab Case

Words are separated using hyphens.

```text
student-profile
main-container
user-details
```

**Note:** Kebab case cannot be used for JavaScript variable names because the hyphen (`-`) is treated as the subtraction operator. It is commonly used for file names, URLs, and CSS class names.

Example:

```html
<div class="main-container"></div>
```
